# 游戏设计大爆炸工具规划

## 最终效果

用户输入一个游戏方向或几个关键词，系统不是一次性生成一个结果，而是像创意进化树一样，带用户一轮一轮做选择。每一轮都围绕一个明确维度爆炸生成多个方案，例如概念、风格、玩法、剧情、角色地图和最终方案。用户每轮选择一个或多个结果，下一轮基于这些选择继续发散。最后系统把从第一轮到最后一轮的所有选择、图片、设定和设计内容串起来，生成一份完整的《游戏设计方案网页》，并支持后续导出成 PDF、Markdown、PPT 或视频。

核心目标不是生成几张图，而是从一个模糊关键词，逐步生成一套可展示、可回溯、可继续开发的游戏设计方案。

## 最终产品形态

工具主界面由三个部分组成：

1. 左侧或顶部流程：游戏概念、视觉风格、核心玩法、世界观剧情、角色地图、最终方案。
2. 中间当前轮次：展示上一轮选中的内容、本轮目标、本轮生成结果、选择按钮和进入下一轮操作。
3. 右侧已选路径：展示从关键词到当前轮次的完整选择链路。

最终方案页自动串联以下内容：

- 封面
- 游戏一句话介绍
- 核心卖点
- 玩法循环
- 美术风格
- 世界观
- 主线剧情
- 角色与地图
- 关卡方向
- 系统设计
- UI 页面方向
- MVP 开发范围
- 后续扩展方向

推荐主交付物是可交互网页。视频适合作为后续宣传物，但不适合作为主交付，因为视频不可编辑、不可回溯、不可继续分支。网页可以承载图片、流程、设计文档和后续导出。

## 第一阶段：流程化

目标：把现有大爆炸模块从自由发散改造成游戏设计多轮流程工具。

第一阶段先实现 6 轮 MVP：

1. 游戏概念
2. 视觉风格
3. 核心玩法
4. 世界观剧情
5. 角色与地图
6. 最终设计方案

需要完成：

- 固定游戏设计轮次。
- 每一轮显示当前轮次名称、本轮目标、上一轮选择和生成结果。
- 每一轮基于上一轮选中的节点或图片继续爆炸。
- 保存每轮生成内容、用户选择、父级关系和图片切片。
- 最后一轮提供基础版游戏设计方案汇总。

第一阶段完成后的效果：用户可以从关键词一路走到最终基础方案，形成一条完整的游戏设计链路。

## 第二阶段：专业化

目标：把生成内容从可用流程升级为更完整的游戏策划案。

第二阶段扩展到完整 10 轮：

1. 游戏概念
2. 视觉风格
3. 核心玩法
4. 世界观剧情
5. 角色体系
6. 地图区域
7. 关卡设计
8. 系统数值
9. UI 页面
10. 最终方案

每一轮补充专业字段：

- 角色：名字、身份、性格、背景、剧情作用、技能定位、与主角关系、成长或反转。
- 地图：区域名称、视觉主题、敌人类型、主要机制、资源产出、剧情事件、Boss 或挑战。
- 关卡：关卡目标、路线结构、敌人配置、机关设计、奖励分布、剧情触发点、Boss 机制、难度曲线。
- 系统：技能系统、装备系统、道具系统、任务系统、成长系统、货币系统、掉落系统、难度系统、成就系统。
- UI：开始界面、角色选择页、地图页、战斗 HUD、背包页、技能树页、商店页、剧情对话页、结算页。

第二阶段还需要支持：

- 单选和多选混合。
- 回到任意一轮重新生成。
- 从任意轮次创建新分支。
- 最终页面生成结构化游戏设计文档。

第二阶段完成后的效果：系统生成的不只是创意页面，而是一份可以给团队评审和继续开发的游戏策划案。

## 第三阶段：生产化

目标：让工具从内部创意工具升级为可交付、可复用、可协作的游戏设计生产工具。

需要完成：

- 多格式导出：网页分享链接、Markdown、PDF、PPT、图片资源包、JSON 项目数据。
- 项目管理：创建项目、保存项目、复制项目、重命名项目、删除项目、历史版本和分支版本。
- 资产沉淀：概念库、风格库、角色库、地图库、关卡库、UI 页面库、Prompt 库和图片素材库。
- 团队协作：评论、批注、多人选择、版本对比、评审状态、确认和驳回。
- 视频扩展：概念宣传视频、角色展示视频、世界观预告片、玩法演示分镜、关卡 walkthrough。
- Demo 扩展：可点击 UI 原型、简单战斗 Demo、地图探索 Demo、剧情对话 Demo。
- 开发衔接：需求文档、任务拆分、美术需求表、程序需求表、关卡制作清单、角色制作清单、Prompt 清单和开发排期。

第三阶段完成后的效果：工具不仅能帮助用户想游戏，还能把想法沉淀成资产、方案、文档和生产任务，接入后续研发流程。

## 阶段总结

第一阶段：流程化。输入关键词，按固定游戏设计轮次多轮生成，最后得到基础方案。

第二阶段：专业化。扩展到完整游戏策划链路，让每轮生成内容更深、更结构化。

第三阶段：生产化。加入导出、项目管理、资产库、协作和开发衔接能力。
